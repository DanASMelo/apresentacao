Bloco 06

IA e Eficiência

Metadata-first navigation via MCP Server. Menos tokens.

Antes (sem metadata)

  • IA lê TODAS as tabelas para entender o banco
  • Milhares de tokens gastos (caro)
  • Lento
  • Erra por falta de contexto
  • Não escala

Depois (com metadata)

  • IA lê metadata primeiro, consulta só o necessário
  • Dezenas de tokens (barato)
  • Rápido
  • Preciso — sabe as regras antes de consultar
  • Escala com novas tabelas

MCP Server — o guia inteligente da IA

Microsserviço interno que funciona como "guia" da IA. Toda vez que o agente recebe uma pergunta, ele é obrigado a chamar o MCP Server primeiro. O MCP consulta só a tabela ConfiguracaoSistema, devolve um JSON pequeno, e só então a IA monta a SQL.

O que a ConfiguracaoSistema contém

Descrição de tabelas/views

Significado de colunas

Regras de COLLATE (como traduzir entre bancos com idiomas diferentes)

Query templates

Thresholds (limites configuráveis para disparar alertas)

Fluxo em 5 passos

1. PerguntaAgente recebe pergunta do usuário
→
2. MCP ServerChama /get_relevant_metadata (obrigatório)
→
3. JSON ~500 tokensAnálise de intenção + ConfiguracaoSistema
→
4. IA gera SQLQuery direcionada com contexto
→
5. ResultadoPreciso e barato

Nova tabela? Só adiciona uma linha na ConfiguracaoSistema — não mexe em nada do agente.
